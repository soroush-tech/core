## @soroush.tech/design-system@0.1.0

Initial release — the soroush.tech design system extracted from the web app into a
standalone package.

### Features

- A token-driven, consumer-extensible `baseTheme` with the full component set: layout & surfaces
  (View, Flex, Grid, Paper, Card, AppBar, Drawer), content & data display (Typography,
  Link, Image, Icon, Table, CodeBlock, Quote, Avatar, Skeleton), inputs & forms
  (Button, ButtonGroup, TextInput, NativeSelect, Select, Checkbox, Radio, Switch,
  ToggleButton, Form, Field, FormControl), feedback (CircularProgress, LinearProgress),
  and overlay primitives (Modal, Popover, Portal, Backdrop, FocusTrap).
- Headless markdown editor & renderer (`Markdown`): Control, Toolbar, Editor, and
  Preview built on `react-markdown`, `remark-gfm`, and `rehype-highlight`.
- Engine barrel re-exporting `@soroush.tech/styled-system` and Emotion primitives
  (`styled`, `css`, `keyframes`, `useTheme`, `Theme`, `createShouldForwardProp`).
- Consumer-owned theming: `ThemeProvider` provides one theme (built-in dark by
  default); `createTheme(base, overrides)` deep-merges sparse patches; every
  scale is an open interface on `@emotion/react` so palettes, tokens, and whole
  scales extend via declaration merging.
- Overridable component defaults: literal fallbacks (`size = 'md'`,
  `variant = 'outside'`, …) resolve through `theme.defaults` / the provider's
  `defaults` prop.
- Per-component customization via `theme.components` — `defaultProps`, per-slot
  `styleOverrides` with `{ theme, ownerState }` callbacks, and theme-contributed
  `variants` (Button is the reference implementation; augmentable
  `ButtonVariants`). Zero cost for themes that don't use it.

### Packaging

- Dual ESM/CJS build via tsdown with a subpath export for every component
  folder, including nested ones (`@soroush.tech/design-system/Button`,
  `…/Table/TableCell`, `…/Markdown/Editor`, `…/themes`, `…/colors`, …).
- `react`, `react-dom`, `@emotion/react`, and `@emotion/styled` are peer dependencies;
  `@tanstack/react-form` is an optional peer (type-only, used by `Field`).
