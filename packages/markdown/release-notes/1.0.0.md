## @soroush.tech/markdown@1.0.0

First stable release — the headless markdown editor + renderer, extracted from
`@soroush.tech/design-system` into its own package. Ships alongside its
companion `@soroush.tech/design-system@1.0.0`, which it builds on (peer) for
primitives, the `styled` engine, and theming. `0.1.0` was never published;
this is the package's first appearance on npm.

### Features

- Headless, composable markdown parts importable from the barrel: `Control` (state provider),
  `Toolbar` (formatting buttons, heading/code selects, table picker), `Editor` (source textarea
  with Tab-to-indent, selection, and caret restore), and `Preview` (a standalone renderer of
  markdown to design-system primitives, built on `react-markdown`, `remark-gfm`, and
  `rehype-highlight`).
- `CodeBlock` — the syntax-highlighted code renderer with a hover-revealed copy control,
  usable standalone and as `Preview`'s code element. Highlight tokens come from
  `theme.syntax` (`syntaxDark` / `syntaxLight` presets included).
- `Mermaid` — fenced ` ```mermaid ` blocks render as diagrams in `Preview` (#282). mermaid is
  loaded lazily via dynamic import, so documents without diagrams never pay for it; per-theme
  styling goes through `theme.mermaid` (`MermaidThemeVariables`, merged over the defaults).
- Per-component theming: the roots register the `MarkdownEditor`, `MarkdownPreview`,
  `MarkdownToolbar`, and `CodeBlock` slots on `@soroush.tech/design-system/theme`'s
  `ThemeComponents`, so consumers customize defaults and styling through design-system's
  `createTheme` / `theme.components`; `theme.syntax` and `theme.mermaid` extend the same
  augmentation surface.

### Packaging

- Peers: `@soroush.tech/design-system` and `react` / `react-dom` `^19`.
- `react-markdown`, `remark-gfm`, `rehype-highlight`, and `mermaid` are regular
  dependencies — consumers install nothing beyond the peers.
