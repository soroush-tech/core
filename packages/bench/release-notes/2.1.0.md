## @soroush.tech/bench@2.1.0

Adds the primitives a CI gate needs: a pass/fail performance target and a
file-based results export. All new flags are optional — omitting them keeps
2.0.0 behavior exactly. (#293)

### Features

- **`--baseline-case <key>` + `--min-ratio <percent>` — the ratio gate.** After a
  run, every other case's mean time is compared against the named baseline case;
  any case whose speed ratio (`baseline.avg / case.avg`) falls below the target
  percentage fails the run with a non-zero exit, printing which cases fell short
  and by how much. A missing/typo'd baseline key is an error, not a silent pass.
  The two flags must be given together.
- **`--md-file <container-path>` — persist the markdown results table.** Writes
  the same table as `--md` to a file (point it inside a writable `--mount`), so
  CI drivers can read results back out of the sandbox instead of scraping stdout.
