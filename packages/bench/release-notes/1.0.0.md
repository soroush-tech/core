## @soroush.tech/bench v1.0.0 🎉

First stable release — run A/B function and npm-package-version benchmarks inside a
**CPU/RAM-pinned Docker sandbox** for stable, low-noise numbers. Built on
[mitata](https://github.com/evanwashere/mitata).

### Highlights

- **`soroush-bench` CLI** — runs a `*.bench.ts` in a pinned sandbox (`--cpuset` / `--cpus` /
  `--memory` / `--tag` / `--mount`).
- **`defineBench()` or install-free** — export a validated config, or `export default` a plain
  object (`satisfies BenchConfig`) and run with `npx` — no import required.
- **Compare package versions** — alias-install two-or-more npm versions inside the sandbox via
  `packages:`, injected as `ctx.modules`.
- **Rich report** — mitata histograms + boxplot chart + `N× faster` summary + `delta vs
fastest`; `--md` markdown table (avg · p75 · alloc/iter with % vs least · gc/iter · vs
  fastest).
- **`--rounds N`** median-of-N with per-round progress, and **`--gc-inner`** per-iteration GC
  row.
- **`options` block** (`gc` / `rounds` / `warmup` / `sandbox`) as file defaults, each
  overridable by the matching CLI flag; `options.sandbox` is read on the host before
  `docker run`.
- Non-root sandbox, `engines: node >= 18`, 100% test coverage.

### Install

`npm install -D @soroush.tech/bench` — Docker must be running on the host.

Quick try, nothing installed: `npx '@soroush.tech/bench' ./my.bench.ts`

### Upgrading from 0.1.x

- The bin was renamed **`bench` → `soroush-bench`**.
- **`warmup`** moved into the **`options`** block.
